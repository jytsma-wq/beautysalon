## Audit Baseline

- Supported locales are `en`, `ka`, `ru`, `tr`, `ar`, and `he`; Arabic and Hebrew are RTL locales.
- Canonical and hreflang alternates are generated through `src/lib/seo.ts`.
- Global and locale sitemaps are generated through `src/app/sitemap.ts`, `src/app/[locale]/sitemap.ts`, and `src/lib/sitemap-routes.ts`.
- Homepage `BeautySalon` JSON-LD is generated from `src/components/seo/JsonLd.tsx` and shared `siteConfig`.
- Homepage `BeautySalon` schema intentionally excludes `aggregateRating`, review markup, and unverified geo coordinates.
- Existing strong local landing pages: `/{locale}/beauty-salon-batumi`, `/{locale}/botox-batumi`, `/{locale}/dermal-fillers-batumi`, `/{locale}/skin-treatment-batumi`.
- Added/selected local-intent landing pages for this phase: `/{locale}/lip-fillers-batumi`, `/{locale}/acne-treatment-batumi`, `/{locale}/nails-batumi`, `/{locale}/lashes-brows-batumi`.
- Microneedling and chemical peel intent is mapped to existing treatment pages to avoid thin doorway pages.
- Visible keyword-stuffing block `"Search phrases this page answers"` is not rendered by the local SEO template and the exact English label was removed from local SEO data.
- Source-of-truth visible phone: `+995 577 34 57 67`.
- Source-of-truth WhatsApp phone: `+995 577 28 68 55`; this intentionally differs from the main visible phone.
- Existing claim risk to review later: some older site data references product brands, certifications, and medical-style claims. This phase does not add new unverified claims.

## Shared Implementation Rules

- Content must stay consultation-led, suitability-based, and people-first.
- Do not add fake reviews, invented staff, fake before/after images, guaranteed outcomes, or aggregate rating markup.
- FAQ schema may only describe visible FAQ content.
- Service schema may only describe visible, existing services.
- If a topic is too narrow for a full useful page, map it to a stronger existing page instead of creating a thin landing page.
